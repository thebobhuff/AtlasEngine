# Case: Greenfield Feature

## Scenario

A new user-facing feature is being introduced in a sparse or early-stage repository with limited existing architecture.

## What This Should Stress

- discovery quality when codebase context is thin
- non-generic PRD generation despite sparse inputs
- task synthesis without invented code locations
- release and operate phases when the system has minimal prior observability

## Failure Signals

- the workflow pretends the codebase is richer than it is
- implementation planning invents architecture specifics without evidence
- release or operate phases assume monitoring that was never specified
