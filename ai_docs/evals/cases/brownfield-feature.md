# Case: Brownfield Feature

## Scenario

A feature must be added to a real, existing codebase with established constraints, dependencies, and backward-compatibility concerns.

## What This Should Stress

- codebase-aware discovery and planning
- traceability from requirements to tasks and tests
- safe rollout planning and rollback readiness
- whether handoffs keep enough context for downstream agents

## Failure Signals

- weak codebase grounding
- tasks that do not reflect likely integration boundaries
- testing plans that ignore regression risk
- shipping steps that do not account for backward compatibility
